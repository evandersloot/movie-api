Movie API Documentation

The application will be built using Node.js on the server-side of a movie web application.

This interactive web application will allow users to sign up, build/edit a profile, and compile a list of their favorite movies

Once a user is signed up, the application will give users access to information about movie genres, directors, and movies themselve.

URL EndPoints and Methods used in API

Business Logic URL HTTP Method Request body data format Response body data format
Return list of movies to user /movies GET none JSON object holding data about all movies
Return data about a single movie by title to the user /movies/[title] GET none JSON object holding data about a single movie. Includes description, genre, director, image URL.
Return data about a genre by name/title /genres/[name] GET none JSON object holding data about movies listed by specified genre requested.
Return data about a director by name /directors/[name] GET none JSON object holding data about the name of the director requested. Includes bio, date of birth, date of death if applicable.
Allow new users to register /users POST JSON object holding data about the new user to be added. JSON object holding data about the new user that was added, as well as apply a unique id number to the user.
Allow new users to update their user information /users/[username] PUT none Text message stating the update was successful.
Allow users to add a movie to their list of favorites /users/[username]/movies/[movie] PUT JSON object holding data about the movie to be added. Text message stating the movie has been added.
Allow users to remove a movie from their list of favorites /users/[username]/movies/remove/[movie] DELETE none Text message stating the movie has been removed from their favorites.
Allow existing users to delete/unregister their account /users/[username] DELETE none Text message stating the user account is no longer active.
