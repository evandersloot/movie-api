Movie API Documentation

The application will be built using Node.js on the server-side of a movie web application.

The application will utilize MongoDB for storing data and Monngoose for the Business Logic Layer.

The application will be deployed to Heroku.

This interactive web application will allow users to sign up, build/edit a profile, and compile a list of their favorite movies

Once a user is signed up, the application will give users access to information about movie genres, directors, and movies themselves.

URL EndPoints and Methods used in API

Business Logic URL HTTP Method Request body data format Response body data format
Return list of movies to user /movies GET none JSON object holding data about all movies
Return data about a single movie by title to the user /movies/[title] GET none JSON object holding data about a single movie.
e.g.
{
_id: 1
Title: "Star Wars: Episode IV - A New Hope",
Description: "Luke Skywalker joins forces with a Jedi Knight, a cocky pilot, a Wookiee and two droids"
},
Genre {
Name: "Science Fiction",
Description: "Science fiction films are made of speculative fiction that contains imagined elements that don't exist in the real world."
},
Director: {
Name: "George Lucas",
Bio: "George Lucas is an American Director...",
Birth: "1944",
Death: " "
},
ImagePath: "starwars.png",
Featured: true
}
Return data about all genres to the user /genres GET none JSON object holding data about all genres.
Return data about a genre by name/title /genres/[name] GET none JSON object holding data about a specified genre requested.
e.g.
Genre {
Name: "Science Fiction",
Description: "Science fiction films are made of speculative fiction that contains imagined elements that don't exist in the real world."
}
Return data about all directors to the user. /directors GET none JSON object holding data about all directors, including name, bio, birth date, and death date if applicable.
Return data about a director by name /directors/[name] GET none JSON object holding data about the name of the director requested.
e.g.
Director: {
Name: "George Lucas",
Bio: "George Lucas is an American Director...",
Birth: "1944",
Death: " "
}
Return a list of all users /users GET none JSON object holding data about all users, including username, password, email, birthday, favorite movies, and id.
Return a single user /users GET none JSON object holding data about a single user.
e.g.
{
"FavoriteMovies" : [],
_id : "1",
Name : "Dave Smith",
Username : "davesmith",
Password : "ilikemovies",
Email : "davesmith@dave.com",
Birthday : "1972-01-01"
}
Allow new users to register /users POST JSON object holding data about the new user that was added, as well as apply a unique id number to the user. Text message stating the user has successfully been added
Allow users to update their user information /users/[username] PUT JSON object specifying data to update. Text message stating the update was successful.
Allow users to add a movie to their list of favorites /users/[username]/movies/[movieID] PUT JSON object holding data about the movie to be added. Text message stating the movie has been added.
Allow users to remove a movie from their list of favorites /users/[username]/movies/remove/[movieID] DELETE none Text message stating the movie has been removed from their favorites.
Allow existing users to delete/unregister their account /users/[username] DELETE none Text message stating the user account is no longer active.
